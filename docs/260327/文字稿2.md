首先是第一问，典型的送分题。
像屏幕里这样分别考虑4种情况就行。
但同学们要时刻记得：
越是简单题，越要注意细节和规范！

接下来看第二问。
我们要证明甲社区组队方案数满足屏幕上方的递推式。
因为甲社区是每个女生都认识所有男生，
呈现一种高度对称性，
所以我们可以大胆猜测，
这里直接算方案数会更简单。
我们考虑3步走：
1，选出m个女生。
2，选出m个男生。
3，让这m个女生和m个男生随机两两组队。
前两步的方案数都是C n m，
第三步的方案数就是排列数，m的阶乘。
把它们乘起来就是A n m。
求出了A n m，验证递推式就简单了。
像屏幕下方这样提取公因式就证毕了。

第二问还有一种更为复杂的做法。
这里为什么要讲这种做法呢？
不是多此一举吗？
因为这里体现的分类讨论思想会给下一问和后面的拓展启发。
首先，我们从解读A n m的语义入手。
A n减1 m表示少了n号男生和n号女生，但还是组成m队。
A n减1 m减1表示少了n号男生和n号女生，但少组一队。
它们的语义启发我们：
考虑对了n号男生和n号女生是否参加社区比赛分类讨论，
共需要讨论2乘2=4种情况。
1，n号男生和n号女生都不参加。
那么其他2n减2人已经组成m队，方案数A n减1 m。
2，n号男生参加，n号女生不参加。
首先其他2n减2人组成m减1队。
此时女生还有n减m+1人没被选中，但记得排除n号女生，
于是n号男生可挑选组队的女生数为n减m。
总方案数(n减m)乘A n减1 m减1。
3，n号女生参加，n号男生不参加。
和“2”类似，不赘述。
4，n号男生和n号女生都参加。
首先其他2n减2人组成m减1队。
接下来是易错点：不要默认n号男生和n号女生在同一队。
实际上n号女生可以选择拆散一个组好的队伍，
从而和其他参赛男生组队。
所以总方案数m乘A n减1 m减1。
综合这4种情况，我们就成功证明了递推式。

接着看第三问。
为了求出概率，只需要求出B的表达式。
于是我们有这样一个朴素的想法：
先试试直接用组合意义写出通项公式。
碰壁了就考虑子问题的性质，先求递推式再求通项！
考虑枚举被选中的m个女生的编号，
这里有C n m种情况。
因为编号的序关系是无关紧要的，
为了后续讨论方便，我们不妨设这些编号是升序的。
考虑第k个被选中的女生，则她能选择的男生个数为2倍i k减k。
于是我们写出了B的通项公式。
化简这式子似乎不容易，我们还是考虑从B的递推式入手吧！

有了第二问法2的基础，我们依葫芦画瓢，
先解读B n m的语义，再分类讨论。
因为最后两位男生只有n号女生认识，
所以这里我们只需要按n号女生是否参赛来分类：
1，n号女生不参赛，方案数B n减1 m。
2，n号女生参赛。
首先，其他3n减3人组成m减1队。
接下来，和第二问法2的情况4不同的是，
n号女生不能拆散组好的队伍。
所以需要在没被选中的2n减m个男生中选一个组队。
所以总方案数2n减m乘B n减1 m减1。
诶，B的递推式竟然和A的递推式完全一样！

不要高兴得太早！
要记得检查边界情况！
不难算出A n 1和B n 1都是n方，
它俩确实相等，放心了！
那这题就做完了！
顺便提一下，如果你打算法竞赛，
你就会知道，
在组合数学里，可以直接约定
A n 0和B n 0都等于1，
这样就不需要验算B n 1了。

诶，刚刚我们领悟了B n m的组合意义，
那就回顾之前求出的通项公式，
看看我们有没有能力化简它吧！
由前面求出的递推式，
我们不难想到可以把这C n m项分为：
i m小于n和i m等于n这两类。
i m小于n，方案数就是B n减1 m。
i m等于n，把2n减m这个公因子提出来，
我们就成功把通项公式转化为递推式了。

题目做完了，但我们精彩的探索才刚刚开始呢！
我们来进一步深挖刚刚那个数列的组合意义！
在OEIS输入这个数列，我们发现：
1，它表示在所有由1到n组成的排列中，
长度为n减k的上升子序列的数量。
2，它也表示在一个n乘n的国际象棋棋盘上，
放置k个互不相撞的车（rook）的方案数。
不难发现，前面第二问等价于模型1：
有车的行可以认为是被选中的男生，
有车的列可以认为是被选中的女生。
下面介绍看待模型1的3个视角。
视角1：直接求方案数。
和前面第二问法1一样，三步走就OK。
这里不赘述。

视角2对应前面第二问的法2。
设dp i j表示在i乘i的棋盘上放j个车的方案数。
模仿前面第二问的法2来分类讨论：
1，第i行和第i列都没有车。
2，第i行放一个车，但第i列不放车。
相当于i号男生参赛但i号女生不参赛。
为了满足这一点，位置i i不能放车，
所以乘的系数是i减j。
情况3和情况2类似。
4，类似地，第j个棋子有两种选择：
1，放在位置i i，贡献的系数是1。
2，挪动之前放好的位置x y的棋子。
那么第j个棋子放在位置i y，
被挪动的棋子放在位置x i，
贡献的系数是j减1。
综上，成功得到和第二问一样的递推式。

最后是视角3，另一种动态规划。
设dp i j表示棋盘前i行上放j个车的方案数。
同样按第j个车放的位置分类讨论。
1，j个车都放在前i减1行。
2，第j个车要放在第i行。
第i行有n个格子，但已经有j减1个被占用，
所以有n减j+1个格子可选。
相比于视角2，视角3的dp定义的普适性更强，
因为它适用于任意形状的棋盘。

下面我们以前面的第三问为例，
来感受一下视角3的dp定义的强大威力。
棋盘的形状变为：总共n行，
第i行有2i减1个棋子。
我们直接看屏幕里分类讨论的情况2：
第j个车放在第i行。
第i行有2i减1个格子，但已经有j减1个被占用，
所以有2i减j个格子可选。
诶，同学们，你们发现了吗？
前面第三问之所以能求出和第二问一样的递推式，
是因为第二问采用了视角2的dp定义，
第三问采用了视角3的dp定义！
妙哉！

第1、2行的格子数是2，

扩展3：洛谷P1350。
给了一个形状比扩展2还要简单的棋盘。
因为形状太简单，所以这题有两种解法。

法1，直接求方案数。
首先，我们把模型1推广到模型2：
把n乘n棋盘升级为n乘m的棋盘。
这里和前面第二问一样三步走就行：
1，n行中选出k行。
2，m列中选出k列。
3，在k乘k的棋盘上放k个车。
接下来回到原问题。
回顾棋盘形状，我们发现棋盘可以看成由两个矩形组成，
所以这里可以有两个想法：
1，先在c乘d的矩形上放车，再在a乘(b+d)的矩形上放车。
2，先在a乘b的矩形上放车，再在(a+c)乘d的矩形上放车。

假设先在c乘d的矩形上放了i个车，
那么a乘(b+d)的矩形就有i行被占用了，
所以相当于矩形变成了a乘(b+d减i)。
套模型2的公式就求出了总方案数。
想法2类似的，这里不赘述。
