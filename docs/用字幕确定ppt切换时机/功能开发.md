## 通义千问网页端《smv项目-用js脚本确定ppt autoslide的时间间隔》

大佬，你是一名专家前端工程师，精通前端工程化。请叫我hans7。我有一个文字稿，以及用edge-tts生成的语音对应的字幕文件。文字稿分为若干个段落，段落之间用一个空行隔开。举例：

文字稿：

```
句子1-1
句子1-2

句子2-1

句子3-1
句子3-2
```

对应的字幕文件：

```
1
00:00:00.000 --> 00:00:01.000
句子1-1

2
00:00:01.000 --> 00:00:02.000
句子1-2

# ……
```

文字稿的文本和字幕的句子出现顺序一致，大部分句子都是完全相同的，但不保证完全一致。现在我希望读入文字稿，求出每个段落的持续时间，并加上一个2.5s左右的延迟。还是以上面的文字稿为例，假设分别求出句子1-1、句子2-1、句子3-1的起始时间分别为a、b、c，句子3-2（文字稿的最后一句话）的结束时间为d，那么这3个段落的持续时间分别为：b-a、c-b、d-c。但我希望加上的延迟总是不小于2.5s且小于3s，比如：

1. b-a=53.2，则期望输出的持续时间为56（下面记为`up`函数）
2. b-a=53.7，则期望输出的持续时间为56.5。期望输出总是0.5的倍数

为了达成这个目标，我希望你的JS脚本支持在多种输入形式中选一种：

1. 字幕文件必须提供
2. 支持选择提供文字稿文件，或选择提供一个字幕编号文件。字幕编号文件的每一行是一个数字，表示文字稿中每个段落的起始句在字幕文件中的编号

并支持在多种输出形式中选一种：

1. 在控制台输出一个字符串`<AutoSlide :timeList="[0, t1, t2, t3, ...]" />`。以前面的例子为例，期望的`t1, t2, t3 = up(b-a), up(c-b), up(d-c)`
2. 不输出，而是修改用户指定的文件，精确匹配`<!-- <AutoSlide :timeList="[0, 0]" /> -->`这个字符串，把它换成上一点说的字符串`<AutoSlide :timeList="[0, t1, t2, t3, ...]" />`。如果出现多次，只替换第一次出现的

其他技术点：

1. JS脚本将用bun执行
2. 匹配文字稿文本和字幕文本时，需要有简单的模糊匹配能力

在开始开发之前，你可以先询问我一些不够清晰的需求点。

---

1. 符合预期
2. 按独立持续时间生成就行
3. 引入编辑距离（Levenshtein）
4. 1-based，对应字幕的原始序号
5. 你的参数结构是OK的，但希望支持简写的别名
6. 必须完全精确匹配（包括空格和换行）
